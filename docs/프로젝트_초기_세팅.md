# 코리빙(Co-Living) IoT 플랫폼 — 프로젝트 초기 세팅

> [!NOTE]
> 기획안, 기능명세서, ERD(v2.0), 개발 환경 버전 정의서를 기반으로 설계한 **백엔드 + 프론트엔드** 프로젝트 초기 세팅 명세입니다.

---

## 1. 기술 스택

### 1.1 백엔드

| 영역 | 버전 | 비고 |
|---|---|---|
| Java | 21 (LTS) | Spring Boot 3.x 공식 권장 |
| Spring Boot | 3.3.x | Java 21 기준 안정 버전 |
| Gradle | 8.10.x (Groovy DSL) | Wrapper로 버전 고정 |
| PostgreSQL | 16 | Docker 이미지: `postgres:16-alpine` |
| Docker Compose | v2 | `docker compose` 명령어 |
| JWT | jjwt 0.12.6 | `io.jsonwebtoken` |
| API 문서 | springdoc-openapi 2.7.0 | Swagger UI 자동 생성 |
| Mock IoT | WireMock 3.5.4 | HTTP Mock 서버 |

### 1.2 프론트엔드

| 영역 | 버전 | 비고 |
|---|---|---|
| Node.js | 20 (LTS) | Next.js 15 공식 권장 |
| Next.js | 15.x (App Router) | React 기반 풀스택 프레임워크, BFF 역할 포함 |
| React | 19.x | Next.js 15 번들 포함 |
| TypeScript | 5.x | 타입 안정성 확보, 프로젝트 기본 언어 |
| Tailwind CSS | 4.x | 유틸리티 퍼스트 CSS 프레임워크 (@tailwindcss/postcss) |
| UI Library | shadcn/ui & Radix | 접근성 및 공용 UI 뼈대 컴포넌트 (headless) |
| Animation  | framer-motion | 부드러운 페이지 화면 진입 및 마이크로 인터랙션 |
| ESLint | 9.x | Next.js 기본 규칙 (`next/core-web-vitals`) |

> [!NOTE]
> Next.js App Router의 **Route Group** `(괄호)` 기능을 사용하여 역할별 페이지를 논리적으로 그룹화합니다. Route Group은 URL 경로에 영향을 주지 않으므로, `/rooms`, `/my-devices`, `/admin/dashboard` 등 직관적인 URL을 유지하면서 역할별 레이아웃과 인증 미들웨어를 분리할 수 있습니다.

> [!IMPORTANT]
> **🌟 Tailwind CSS v4 환경 구성 원칙**
> 기존 `tailwind.config.js`를 완전히 폐기하고, CSS 인라인 변수 기반의 최신 스펙을 사용합니다.
> - **`src/styles/theme.css`**: UI 가이드라인(Moss & Aloe 등)에 정의된 컬러칩, 간격, 폰트를 `:root`와 `@theme inline` 지시어로 매핑합니다.
> - **Prettier & ESLint**: `.prettierrc`와 `eslint.config.js`를 통해 Tailwind 클래스 네임 자동 정렬 및 문법 검사를 강제합니다. 향후 AI에게 컴포넌트 생성을 요청할 때 절대 Hex코드를 하드코딩하지 말고 `theme.css`에 선언된 유틸리티 클래스만 사용하도록 해야 합니다.

> [!IMPORTANT]
> `(resident-app)/` 그룹은 **모바일 지향 UI**로 설계합니다. 반응형 레이아웃과 터치 인터랙션을 최우선으로 고려하며, 하단 내비게이션 바(BottomNavBar)를 사용합니다.

## 4. 환경 변수 분리

### 4.1 application.yml (공통)

```yaml
spring:
  profiles.active: dev
  jpa:
    hibernate.ddl-auto: validate
    open-in-view: false
    properties.hibernate.format_sql: true
  jackson:
    time-zone: Asia/Seoul
    property-naming-strategy: SNAKE_CASE
  servlet:
    multipart:
      max-file-size: 15MB        # 개별 파일 제한 (여유있게 설정)
      max-request-size: 50MB     # 전체 요청 크기 제한 (다중 업로드 감안)

server:
  port: 8080

springdoc:
  swagger-ui.path: /swagger-ui.html
```

### 4.2 application-dev.yml (개발)

```yaml
spring:
  datasource:
    url: ${DB_URL:jdbc:postgresql://db:5432/coliving}
    username: ${DB_USERNAME:coliving}
    password: ${DB_PASSWORD:coliving123}
  jpa:
    show-sql: true
    hibernate:
      ddl-auto: update   # 개발 시에는 Entity 기준으로 테이블 자동 생성/변경 (사용자 피드백 반영)
  sql.init.mode: never   # JPA가 자동 생성하므로 schema.sql 직접 실행 끄기

jwt:
  secret: ${JWT_SECRET:dev-secret-key-minimum-32-characters-long!!}
  access-expiration: 1800000       # 30분
  refresh-expiration: 604800000    # 7일

mock-iot:
  base-url: ${MOCK_IOT_URL:http://localhost:8081}

logging:
  level:
    com.coliving: DEBUG
    org.hibernate.SQL: DEBUG
```

### 6.3 BFF 프록시 — `route.ts`

브라우저에서 `/api/bff/...`로 요청하면 Next.js 서버가 Spring 백엔드로 프록시합니다.
JWT 토큰은 httpOnly 쿠키에서 추출하여 `Authorization` 헤더로 변환합니다.

```typescript
// src/app/api/bff/[...path]/route.ts
import { cookies } from 'next/headers';
import { NextRequest, NextResponse } from 'next/server';

const BACKEND_URL = process.env.INTERNAL_BACKEND_URL || 'http://localhost:8080';

async function handler(req: NextRequest, { params }: { params: Promise<{ path: string[] }> }) {
  const { path } = await params;
  const backendPath = `/api/${path.join('/')}`;
  const url = `${BACKEND_URL}${backendPath}${req.nextUrl.search}`;

  // httpOnly 쿠키에서 JWT 추출
  const cookieStore = await cookies();
  const accessToken = cookieStore.get('access_token')?.value;

  const headers: HeadersInit = {
    'Content-Type': req.headers.get('Content-Type') || 'application/json',
  };
  if (accessToken) {
    headers['Authorization'] = `Bearer ${accessToken}`;
  }

  const response = await fetch(url, {
    method: req.method,
    headers,
    body: ['GET', 'HEAD'].includes(req.method) ? undefined : await req.text(),
  });

  return new NextResponse(response.body, {
    status: response.status,
    headers: { 'Content-Type': response.headers.get('Content-Type') || 'application/json' },
  });
}

export const GET = handler;
export const POST = handler;
export const PUT = handler;
export const PATCH = handler;
export const DELETE = handler;
```

### 6.5 BFF fetch 래퍼 — `api.ts`

```typescript
// src/lib/api.ts
import { ApiResponse } from '@/types/api';

const BASE_URL = process.env.NEXT_PUBLIC_API_URL || '/api';

export async function apiFetch<T>(
  path: string,
  options?: RequestInit
): Promise<ApiResponse<T>> {
  const response = await fetch(`${BASE_URL}/bff${path}`, {
    ...options,
    headers: {
      'Content-Type': 'application/json',
      ...options?.headers,
    },
    credentials: 'include',   // httpOnly 쿠키 자동 전달
  });

  const data: ApiResponse<T> = await response.json();

  if (!data.success) {
    throw new ApiError(data.message || '요청에 실패했습니다', data.error_code);
  }

  return data;
}

export class ApiError extends Error {
  constructor(
    message: string,
    public errorCode?: string
  ) {
    super(message);
    this.name = 'ApiError';
  }
}
```

### 8.3 요청 추적 — `CorrelationIdFilter`

- 모든 요청에 UUID 기반 `X-Correlation-Id` 부여

- `CONTROL_LOG.correlation_id`에도 기록
