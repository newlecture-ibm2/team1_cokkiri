# 코리빙(Co-Living) IoT 플랫폼 — 프로젝트 초기 세팅

> [!NOTE]
> 기획안, 기능명세서, ERD(v2.0), 개발 환경 버전 정의서를 기반으로 설계한 백엔드 프로젝트 초기 세팅 명세입니다.

## 1. 기술 스택

| 영역 | 버전 | 비고 |
|---|---|---|
| Java | 21 (LTS) | Spring Boot 3.x 공식 권장 |
| Spring Boot | 3.3.x | Java 21 기준 안정 버전 |
| Gradle | 8.10.x (Groovy DSL) | Wrapper로 버전 고정 |
| PostgreSQL | 16 | Docker 이미지: `postgres:16-alpine` |
| Docker Compose | v2 | `docker compose` 명령어 |
| JWT | jjwt 0.12.6 | `io.jsonwebtoken` |
| API 문서 | springdoc-openapi 2.7.0 | Swagger UI 자동 생성 |
| Mock IoT | WireMock 3.5.4 | HTTP Mock 서버 |

## 6. 공통 처리 로직

### 6.1 통일된 응답 포맷 — `ApiResponse<T>`

```java
public class ApiResponse<T> {
    private boolean success;
    private T data;
    private String message;
    private String errorCode;     // 실패 시만

    public static <T> ApiResponse<T> ok(T data) { ... }
    public static <T> ApiResponse<T> ok(T data, String message) { ... }
    public static ApiResponse<?> error(ErrorCode code) { ... }
}
```

**성공 응답 예시:**
```json
{ "success": true, "data": { "user_id": 1, "login_id": "user01" }, "message": null }
```

**실패 응답 예시:**
```json
{ "success": false, "data": null, "message": "이미 사용 중인 로그인 ID입니다", "error_code": "DUPLICATE_LOGIN_ID" }
```

### 6.2 전역 예외 처리 — `GlobalExceptionHandler`

| 예외 | HTTP | 처리 |
|---|---|---|
| `BusinessException` | ErrorCode 정의값 | `ApiResponse.error(errorCode)` |
| `MethodArgumentNotValid` | 400 | 필드별 검증 에러 목록 |
| `AccessDeniedException` | 403 | 권한 부족 메시지 |
| `HttpRequestMethodNotSupported` | 405 | 지원하지 않는 HTTP 메서드 |
| `Exception` (그 외) | 500 | 서버 내부 오류 |

## 7. 로깅 및 보안

### 7.1 Spring Security 접근 제어

```
permitAll:
  POST /api/auth/register           회원가입
  POST /api/auth/login              로그인
  POST /api/auth/refresh            토큰 갱신
  GET  /api/rooms/**                방 둘러보기
  GET  /swagger-ui/**, /v3/api-docs/**  API 문서

hasRole("ADMIN"):
  /api/admin/**                     관리자 API 전체

hasAnyRole("RESIDENT", "ADMIN"):
  /api/devices/**                   기기 제어
  /api/facilities/**                시설 조회
  /api/reservations/**              예약
  /api/control-logs/**              제어 이력

authenticated:
  그 외 /api/**                     로그인 필수
```

### 7.2 JWT 인증 흐름

```
요청 → CorrelationIdFilter (추적 ID 부여, MDC 저장)
     → JwtAuthenticationFilter (토큰 검증 → SecurityContext 설정)
     → Controller
```

**JWT Payload (RESIDENT):**
```json
{
  "sub": "1",
  "role": "RESIDENT",
  "contract_id": 5,
  "space_id": 3,
  "exp": 1711900000,
  "iat": 1711898200
}
```

### 7.3 요청 추적 — `CorrelationIdFilter`

- 모든 요청에 UUID 기반 `X-Correlation-Id` 부여
- SLF4J MDC에 저장 → 모든 로그에 자동 포함
- `CONTROL_LOG.correlation_id`에도 기록

```
[2026-03-31 14:00:00] [corr:abc123] INFO ContractService - 계약 체결: userId=1, spaceId=3
```

### 7.4 Mock IoT 연동 — `MockIotClient`

- `RestTemplate`으로 Mock IoT 서버에 HTTP POST 호출
- 기기 제어 명령을 JSON으로 전달, 응답을 SUCCESS/FAILURE로 반환
- URL: `mock-iot.base-url` 설정에서 로드
- 실제 IoT 교체 시 이 클래스만 수정
